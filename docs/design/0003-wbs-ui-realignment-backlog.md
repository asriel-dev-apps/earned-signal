# Design 0003: WBS UI 実スプシ準拠への修正バックログ

Status: ユーザーフィードバック集約（2026-07-21）。advisor 整理済み・実装前。
原則: **実スプシが唯一の正。指示のない列・UI・機能を追加しない**（今回の反省: レビュー管理No/変更管理/重み 列を無断追加していた）。

---

## A. 認証・アクセス

### A-1. 未ログインでも見れて触れてしまう
- 現状: 未ログイン時に合成デモ（PREVIEW）が表示され、編集も可能（ブラウザ内のみ）。
- 修正: **未ログイン時はログイン画面のみ**とし、WBS は認証後にのみ表示する。
- 併せて削除: PREVIEW モード、「デモに戻す」ボタン、preview 用 localStorage 永続化。
- （開発用デモが必要なら env フラグでローカル限定にする。公開環境では出さない。）

## B. 列・ヘッダのスプシ整合

### B-1. スプシに無い列の削除
- 削除: **レビュー管理No**・**変更管理**・**重み**。
- 総点検: 上記以外にもスプシに無い列・装飾が無いか洗い出して除去する（ロック列は C-2 参照）。

### B-2. 見積り 2列
- **工数(人時) = 手入力**（見積の入力箇所。これが BAC の元になる）。
- **工数(人日) = 自動**（人時/8、入力不可）。

### B-3. 計算セルは入力不可
- 関数で算出される列（工数(人日)/計画工数/計画進捗工数/進捗率(計画)/開始予定/終了予定/ステータス/実績進捗工数/コスト差異 など）は**編集不可**にし、入力可セルと視覚的に区別する。
- 入力可: 工数(人時)（見積）、進捗率%（実績）、実績投入工数、開始日/終了日（実績）、メタ入力列、日別セル（C-2 の条件下）。

### B-4. ヘッダをスプシの形に合わせる
- 現状の「上部 KPI タイル + 単段ヘッダ」をやめ、スプシ同様の **2段グループヘッダ**にする:
  - グループバンド: **見積り / BAC / PV / EV / AC / CV** をスプシの配色（緑=PV系、黄=EV系、橙=AC、マゼンタ=CV 等）で色分けし、その下に各列名。
- プロジェクト合計（人日: BAC PV EV AC SV CV SPI CPI）は、スプシのようにグリッド上部の**小さな合計表**として表示（現 KPI タイル列は廃止）。

### B-5. 日付ヘッダ・非稼働日
- 日付ヘッダもスプシ準拠（**月バンド + 日**の2段）。
- **土日・祝日はセルをグレーアウトし入力不可**。
- **有給（個人の非稼働日）は別色**で表示し、入力不可。
- （スケジューラも非稼働日には配置しない=既存挙動と整合）

## C. グリッド操作

### C-1. 表示モード
- **フラット表示を削除**し、ツリー表示のみにする（切替 UI 撤去）。

### C-2. ロック列の扱い
- 経緯: 日別セルは既定でスケジューラ自動配置。行を「手動ロック」すると自動配置対象外になり日別セルを手入力できる、という D17 実装の UI として追加した列。**スプシには存在しない**。
- 修正: **ロック列（左端の鍵アイコン列）を廃止**。挙動は「**日別セルを直接編集したらその行は自動的に手動扱い**（内部フラグ）」に変更し、UI 上の専用列は持たない。
  - 自動配置に戻す操作（フラグ解除）は行コンテキストメニュー等の目立たない場所に置く。

### C-3. 行ドラッグ（⠿）による移動
- **⠿ ハンドルは行の左端**に置く。全行でドラッグ可能。▲▼ボタンは廃止。
- 仕様:
  - **サブタスク行**: 同じ親の領域内で上下に移動（並び替え）。**親の領域から出たらその操作は無効**。
  - **親タスク行**: サブツリーごと移動。別の親タスクの**上 or 下**に落とすと、**その位置に兄弟として**入る（配下には入らない）。
- 未決（ユーザー確認）: サブタスクを**別の親へ付け替える**操作は残すか？ 残す場合の操作方法（例: ドラッグではなく編集/メニューで親を選択）。

### C-4. タスク追加（スプシ流）
- 「タスク追加」ボタンをやめ、**末尾の空行に直接入力して追加**する方式に。
- **「+n 行追加」**機能（n は 1〜1000）で末尾に空行を足せる。空行は名前等を入力した時点で実タスク化。

### C-5. サブタスク追加・テンプレ適用
- タスク（親）を作ると**その配下がサブタスク追加モード**になり、スプシのように自由に行を足せる。
- テンプレ適用はツールバー上のボタン/セレクタを**廃止**し、行に紐づく UI に変更。候補（要選定）:
  - a) 親行の右クリック/⋯メニューに「テンプレから生成」
  - b) 親直下の空サブタスク行に入力補助（「テンプレを適用…」の候補チップ）
  - c) 親行ホバー時の小さな ⊕ ボタン → テンプレ選択ポップ
- 生成後の再按分などの既存ロジックは維持（重み列は出さない。内部値として保持）。

### C-6. ドロップダウン入力
- **工程 / プロダクト / 担当** はドロップダウン選択にし、マスタ（E-2）に無い値は入れられない（バリデーション）。

### C-7. ツリー折畳時の親行ロールアップ
- **折畳時**: 親タスク行に配下の**総工数**と、日別列に**各日の合計**を表示。
- **展開時**: 親タスク行は何も表示しない（現仕様どおり）。

## D. ツールバー等の削減

### D-1. 削除する領域
- 「フラット/ツリー」切替と並べ替えヒントの行（C-1/C-3 により不要）。
- 「タスク追加 / デモに戻す / テンプレート / サブタスク生成」のツールバー行（A-1/C-4/C-5 により不要）。
- 「他PJ負荷: 表示/非表示」トグルと説明文の行。
  - 機能自体（半透明の他PJ負荷・キャパ超過の赤表示）は維持し、**常時表示 or 設定メニューに格納**（要選定）。凡例はツールチップ/ヘルプへ。

## E. 画面追加（マスタ・テンプレ管理）

### E-1. サブタスクテンプレート 一覧・編集画面
- テンプレの一覧/作成/編集/削除。項目: 名称、サブタスク構成（名称・重み%・依存/ラグ）。
- 現状のビルトイン定数をやめ、**DB マスタ化**（プロジェクト or テナント単位、要決定）。

### E-2. マスタ 一覧・編集画面
- **工程 / プロダクト**（＋担当メンバー）の一覧・編集。C-6 のドロップダウンの供給源。
- 現状 text 自由入力の process/product を**マスタ参照**に移行（スキーマ変更含む）。

## F. 採番・連携

### F-1. No. のユニーク化（工数入力・勤怠システム連携を見据える）
- 要件: No. をユニークにしたい。スプシは「pj名-0001」形式だが、参照しやすさ・更新しやすさで最適化したい。タスク/サブタスクの区別は不要か？も要検討。
- 提案（advisor 案・要承認）:
  - **内部キー = 既存 UUID**（不変・グローバルユニーク。外部システム連携・API 参照はこれを正とする）。
  - **表示/人間用 No. = プロジェクト内の不変連番**（作成時に採番、並び替え・削除で振り直さない=欠番許容。例 `0001`。プロジェクト横断で書くときは `PJコード-0001`）。
  - 行の表示順は No. と独立（sort_order）。タスク/サブタスクは**同一の採番体系**（区別しない。階層は parent 関係が持つ）。
  - DB: projects に連番カウンタ（または sequence）、tasks に `seq` 列（tenant,project,seq unique）。

## G. メンバー日次合計の可視化

### G-1. その日にメンバーへ合計何 h 振られているか
- スプシでは日付ヘッダ上にメンバー名と合計工数が並ぶ。より良い UI を提案（候補・要選定）:
  - a) **下部固定パネル**: 行=メンバー、列=同じ日付軸の「日次合計ヒートマップ」（合計h表示、キャパ超過は赤）。トグルで開閉。グリッドと横スクロール同期。
  - b) 日付ヘッダ直下に**選択メンバーのみ**の合計行を挿入（担当セルクリックでそのメンバーを選択）。
  - c) メンバー列のホバー/クリックで**ポップオーバー**にその人の日別合計ミニチャート。
- 他PJ負荷（既実装の ExternalLoad）も合算値に含める。

## 実装順の提案（advisor）
1. **P0 スプシ整合**: B-1 列削除 → B-2/B-3 見積・入力可否 → B-4/B-5 ヘッダ/日付/非稼働日 → C-2 ロック列廃止 → D-1 ツールバー削減 → A-1 認証必須
2. **P1 操作系**: C-4 末尾行追加 → C-5 サブタスクモード/テンプレUI → C-3 ドラッグ移動 → C-6 ドロップダウン → C-7 折畳ロールアップ
3. **P2 画面/マスタ**: E-2 マスタ画面（＋スキーマ） → E-1 テンプレ画面 → C-6 の完全マスタ連動
4. **P3 連携基盤**: F-1 採番 → G-1 メンバー日次合計

## ユーザーに確認したい点（実装前）
1. C-2: ロック列廃止＋「日別セル直接編集で自動的に手動扱い」の方針で良いか。
2. C-3: サブタスクの**親付け替え**は必要か。必要ならドラッグ以外のどの操作にするか。
3. D-1: 他PJ負荷の表示は「常時」or「設定メニューでON/OFF」どちらが良いか。
4. E-1/E-2: テンプレ・マスタは**テナント共通**か**プロジェクト単位**か。
5. F-1: 採番提案（UUID内部キー＋プロジェクト内不変連番、タスク/サブタスク同一体系）で良いか。
6. G-1: メンバー日次合計 UI は a/b/c どれが良いか（推奨は a の下部固定パネル）。
